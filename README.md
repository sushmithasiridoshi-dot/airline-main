# SkyForecast — Airline Passenger Forecasting

Streamlit app that uses an LSTM to forecast future monthly airline
passenger volume.

## Project structure

```
airline-passengers/
├── app.py                  # Streamlit front-end (entry point)
├── requirements.txt
├── runtime.txt              # best-effort Python version pin (see note below)
├── .streamlit/
│   └── config.toml          # dark theme base
├── Data/
│   └── airline-passengers.csv
├── models/                  # put lstm_model.keras + scaler.pkl here
├── outputs/                 # metrics.txt / PNGs get written here
└── src/
    ├── __init__.py
    ├── data_loader.py
    ├── preprocessing.py
    ├── sequence_generator.py
    ├── train_test_split.py
    ├── model.py
    ├── train.py
    ├── predict.py
    ├── evaluation.py
    └── visualization.py
```

**Push the whole folder to GitHub — not just `app.py`.** `app.py`
imports the `src` package and reads `Data/airline-passengers.csv` and
`models/lstm_model.keras` / `models/scaler.pkl` at runtime, so those
have to exist in the repo too.

## Before you deploy: get the model files into `models/`

The app needs two files that aren't code — they're generated by
training:

- `models/lstm_model.keras`
- `models/scaler.pkl`

If you already have them from a previous run, just drop them into
`models/`. If not, run once locally:

```bash
pip install -r requirements.txt
python -m src.train
```

That trains the model and writes both files into `models/`. Commit
them to the repo (they're small — a few MB — so plain git is fine, no
Git LFS needed).

## Why your last deploy failed

The log showed:

```
Because tensorflow==2.16.2 has no wheels with a matching Python ABI
tag and you require tensorflow==2.16.2 ...
Using Python 3.14.6 environment at /home/adminuser/venv
```

TensorFlow only ships wheels for Python 3.9–3.12. Streamlit Community
Cloud has a known, currently-open issue where it silently ignores
`runtime.txt` and defaults new/redeployed apps to Python 3.14 anyway
— other people hit this exact TensorFlow failure for the same reason.
`runtime.txt` (included here) is a harmless fallback, but **don't rely
on it alone.**

**Do this instead, every time you deploy or redeploy:**

1. On share.streamlit.io, click **Deploy an app** (or, for an
   existing app, **⋮ → Settings → Advanced settings**).
2. In **Advanced settings**, explicitly pick **Python 3.11** (or
   3.12) from the dropdown before deploying.
3. Note: if the app is already deployed on the wrong version, you
   can't change it in place — delete the app and redeploy fresh with
   3.11 selected.

## Also fixed in this version

The original `src/` files had absolute paths hardcoded to one
machine, e.g.:

```python
self.data_path = "/Users/bouroju/Documents/teck-works/.../airline-passengers.csv"
```

That works on the original author's laptop and nowhere else —
including a fresh clone on Streamlit Cloud. Every module now resolves
paths relative to the project root (`Path(__file__).resolve().parent.parent`),
so it works identically locally, on a teammate's machine, or on
Streamlit Cloud. `models/` and `outputs/` are also created
automatically if missing, so `evaluation.py`'s metrics file and
`preprocessing.py`'s saved scaler don't fail on a clean checkout.

## Local run

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Deploying on Streamlit Community Cloud

1. Push this whole folder to your GitHub repo (root of the repo, or
   note the subfolder as the app's working directory in settings).
2. On share.streamlit.io: **New app** → pick the repo/branch → main
   file path `app.py`.
3. **Advanced settings → Python version → 3.11** (see above — don't
   skip this step).
4. Deploy. First build takes a few minutes since TensorFlow is a
   large dependency.
